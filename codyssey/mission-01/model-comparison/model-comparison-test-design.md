# 모델 비교 테스트 설계 (메일 초안 작성 과업)

## 1. 테스트에 사용할 동일 입력 프롬프트

아래 프롬프트를 3개 이상의 모델(예: ChatGPT, Claude, Gemini)에 **그대로 동일하게** 입력합니다.

```
[업무 과업] 메일 초안 작성

[목적] 외부 협력사에 프로젝트 일정이 1주일 지연됨을 알리고 양해를 구하기

[대상] 협력사 담당자 (직급/이름 모름, 공식적인 관계)

[상황] 내부 개발 일정 조율 문제로 기존 약속한 1차 산출물 제출일이 1주일 늦어지게 됨

[핵심 포인트]
- 지연 사실과 새로운 일정(1주일 후) 명시
- 지연에 대한 사과
- 향후 일정에 차질 없도록 노력하겠다는 의지 표현

[톤] 정중하고 격식 있는 비즈니스 톤

[금지어] "어쩔 수 없이", "사정상" 같은 모호한 책임 회피 표현

[필수 포함 항목] 제목, 인사말, 본문(지연 사실+사과+새 일정), 마무리 인사
```

## 2. 평가 축 (최소 4개 + 점수 기준)

각 모델 결과물을 1~5점으로 채점합니다.

| 평가 축 | 설명 | 1점 | 5점 |
|---|---|---|---|
| ① 형식 준수 | 제목/인사말/본문/마무리 구조를 빠짐없이 지켰는가 | 구조 누락 多 | 완벽히 준수 |
| ② 톤 적합성 | 정중하고 격식 있는 비즈니스 톤인가 | 캐주얼/어색함 | 자연스럽고 격식 있음 |
| ③ 지시사항 반영 | 핵심 포인트 3가지를 모두 담았는가, 금지어를 피했는가 | 누락/금지어 사용 | 모두 반영 |
| ④ 한국어 자연스러움 | 번역체나 어색한 표현 없이 자연스러운가 | 직역체/어색 | 매끄러운 한국어 |
| ⑤ (선택) 추가 디테일 | 별도 지시 없이도 적절한 추가 제안(예: 첨부 안내)을 했는가 | 없음 | 있음 |

## 3. 기록할 항목 (재현성 + 보고서용)

모델별로 아래를 기록합니다.

- 모델명 (예: GPT-4o, Claude Sonnet 4.6, Gemini 2.5 Pro)
- 사용 환경 (웹/앱/API)
- 요금제 (무료/유료) — 무료라면 제한점 1줄
- 사용 날짜 (YYYY-MM-DD)
- 생성된 메일 초안 전문
- 평가 축별 점수 + 한 줄 근거

## 4. 다음 단계

1. 위 프롬프트를 3개 모델에 입력
2. 각 모델 출력 전문을 그대로 복사해서 가져오기 (이 폴더에 `model-outputs-raw.md`로 저장 추천)
3. 위 평가 기준으로 점수표 작성 → `model-comparison.md` (최종 보고서) 완성
