## 3. 과제 목표 (평가자에게 설명 가능해야 하는 항목)

- [ ] 텍스트/이미지/비디오/오디오 생성 도구의 역할과 차이(강점/약점)를 말로 설명할 수 있다.
- [ ] 기획 의도에 맞는 결과물을 얻기 위해 프롬프트를 어떻게 설계·수정했는지 설명할 수 있다.
- [ ] 동일한 요구를 여러 도구로 시도했을 때 결과 차이를 비교하고, 도구 선택의 이유를 설명할 수 있다.
- [ ] 생성된 미디어 소스를 해상도/비율/톤앤매너 관점에서 정리하고, 하나의 광고 영상으로 통합하는 흐름을 설명할 수 있다.

## 4. 기능 요구 사항

- [x] 브랜드(가상 또는 실제) 1개를 선정하고 타겟/톤앤매너/차별점(USP)을 포함한 브랜드 아이덴티티를 정의한다. → CraftFit (`storyboard/storyboard.md` 1번)
- [x] 광고의 목적(인지/전환/리텐션 등)과 핵심 메시지 1개를 한 문장으로 명확히 정의한다. → `storyboard/storyboard.md` 2번
- [x] 영상 제작 전 씬 단위로 스토리보드를 작성하고 씬별 필수 필드를 누락 없이 채운다. → 이미지·비디오·오디오 필드 전부 완료(`storyboard/storyboard.md` 5·7·8번)
- [x] 최소 1개 씬에서 프롬프트를 "수정 전/후"로 비교하고 무엇을 왜 바꿨는지(의도/결과)를 기록한다. → 씬 4 (v1→v2)
- [x] 이미지 생성 AI 1종 이상, 비디오 생성(또는 변환) AI 1종 이상, 오디오 생성 AI 1종 이상을 사용한다. → 이미지(GPT Image 2)·비디오(Google Vids/Veo 3.1)·오디오(ElevenLabs) 전부 완료
- [x] 도구는 자유롭게 선택하되 각 도구를 "왜 그 목적에 선택했는지"를 스토리보드에 간단히 남긴다. → `storyboard/storyboard.md` 4번
- [x] 최종 영상은 10초 이내로 제작한다. → 정확히 10.000초
- [x] 영상에는 AI로 생성된 시각 요소(이미지/영상)와 청각 요소(음성/효과음/배경음악 중 1개 이상)가 반드시 포함되어야 한다. → 시각(GPT Image 2→Google Vids 영상)+청각(ElevenLabs 나레이션)
- [x] 단순 컷 나열이 아니라 최소한 다음 중 하나를 만족하는 스토리라인을 구성한다 — 기승전결(문제 제시 → 전환 → 해결/제안) 구조 → 씬1(문제 제시)→씬2(전환)→씬3(해결)→씬4(브랜드 인지)
- [x] 또는 명확한 메시지 전달("누구에게 어떤 가치를 주는지")이 드러나는 구조
- [x] 영상의 마지막 3~5초 구간에는 브랜드/제품을 인지할 수 있는 장치(로고, 브랜드명, 슬로건, CTA 중 1개 이상)를 포함한다. → 씬4(7.07~10.00초)에 "CraftFit" 워드마크 노출

## 5. 보너스 과제 (추가 점수 목적으로 기본 수행 대상 — 사용자 지침)

- [x] 보너스 1 – 립싱크(Lip-sync) 적용: 인물 발화 장면 1개 이상을 추가하고, 입모양과 대사를 자연스럽게 맞춘다. → GPT Image 2(인물 사진) + ElevenLabs(음성) + Magic Hour Talking Photo(립싱크)로 제작(`storyboard.md` 10번, `storyboard/bonus1/`)
- [x] 보너스 2 – 동일 스토리보드, 다른 도구로 재제작: 이미지 또는 비디오 생성 파트를 다른 도구로 바꿔 동일 씬 1~2개를 재제작한다. → 씬1을 Sora 2 Pro·Google Vids 두 도구로 재제작하고 비교(`storyboard.md` 11번 섹션)
- [x] 보너스 3 – 플랫폼별 화면 비율 버전 제작: 동일 스토리보드로 화면 비율 버전 2개 이상을 제작한다. (예: 16:9(유튜브) + 9:16(숏폼/릴스 표준, 1080x1920) + 1:1(피드용)) → 16:9(필수)+9:16+1:1 3종 전부 제작 완료(`storyboard.md` 12번, `CraftFit_ad_final_9x16.mp4`/`CraftFit_ad_final_1x1.mp4`)

## 6. 제약 사항

- [x] 직접 촬영한 영상, 유료 스톡 영상 소스는 사용하지 않는다. → 전부 생성형 AI(GPT Image 2/Google Vids/ElevenLabs/Magic Hour), 실촬영·스톡 소스 없음
- [x] 모든 시각 및 청각 소스는 생성형 AI를 통해 만들어진 결과물이어야 한다. → 위와 동일
- [x] 저작권·윤리 이슈가 큰 딥페이크, 혐오/선정성/폭력 등 유해 콘텐츠 생성은 금지한다. → 보너스1 인물도 AI 생성 가상 인물(실존 인물 아님), 유해 콘텐츠 없음
- [x] 영상 편집 소프트웨어(Premiere Pro, CapCut 등)는 컷 편집, 자막, 간단한 색보정/오디오 레벨 조정 등 "통합 편집" 용도로만 제한적으로 사용한다. → ffmpeg로 컷 편집·오디오 교체·크롭만 수행(새 AI 생성 없음)
- [x] 영상의 핵심 비주얼/오디오는 AI 생성 결과물을 주된 소스로 유지해야 한다. → 핵심 비주얼/오디오 전부 AI 생성물 그대로 사용
- [x] 영상 생성은 크레딧 소모가 큰 경우가 많으므로, 이미지 생성 단계에서 스토리보드를 최대한 확정한 뒤 영상 변환을 시도한다. → 5번 섹션(이미지) 확정 후 7번 섹션(영상) 진행한 순서로 실제 작업
- [ ] 크레딧이 부족해질 경우, 아래 중 최소 1개 전략을 선택해 범위를 조정한다 — 씬 수를 줄이고(예: 6씬 → 4씬) 메시지 밀도를 높이기 → 해당 없음(크레딧 부족 상황 자체가 발생하지 않음 — 4씬은 크레딧 이슈가 아닌 최초 설계)
- [ ] 크레딧 부족 대응 전략 — 정지 이미지 + 짧은 모션(패닝/줌/루프) 중심으로 구성하기 → 해당 없음(위와 동일)
- [x] 크레딧 부족 대응 전략 — 동일한 스타일 레퍼런스를 고정해 재생성 횟수 줄이기 → 크레딧 문제와 무관하게 일관성 목적으로 이미 적용(5번 섹션 "공통 스타일 앵커")
- [x] 특정 도구가 대기열/플랜 제한으로 막힐 수 있음을 전제로, 이미지/비디오/오디오 각각 대체 도구를 1개 이상 준비한다 (도구명만 기록해도 됨). → **최초 누락 발견 후 추가**: 4번 섹션 "대체 도구 준비" 표(이미지: Midjourney/Firefly, 영상: Sora 2 Pro(실사용 이력), 음성: Typecast/TTS-1 HD(실사용 이력))
- [x] (권장) 캐릭터 및 스타일 일관성 유지를 위해, 도구가 제공하는 Character Reference/Style Reference 기능(예: cref, sref) 또는 ControlNet 계열 제어 기능을 적극 활용해본다. → GPT Image 2에는 cref/sref류 기능이 없어, 대신 모든 씬 프롬프트에 동일한 "공통 스타일 앵커" 텍스트를 고정 삽입하는 방식으로 대체 적용(5번 섹션)
- [x] (권장) 사용했다면, 스토리보드 또는 발표에서 "어떤 파라미터/레퍼런스를 고정했는지"를 설명할 수 있어야 한다. → 색상 팔레트 코드(#8FA998 등)·타이포·레이아웃 서술을 고정 문구로 5번 섹션에 기록
